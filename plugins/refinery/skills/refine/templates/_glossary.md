## How This Glossary Is Maintained

- **Init:** `/refine init` discovers domain terms from filenames and directory names; adds them with empty definitions
- **Authoring:** `spec-writer` agent (per its system prompt §"Memory Usage" and §"Writing Style") consults this glossary on every spec authoring; populates definitions when authoritative
- **Iteration:** `code-archaeologist` may discover new terms during research; surfaces them via spec-scribe to be added here
- **Manual:** users may edit this file directly; conventions per §11 of `_conventions.md`

## Disambiguation

When a term has multiple meanings:

```markdown
| auth (subsystem) | The authentication subsystem; see also: design.md §1 |
| auth (token) | A short-lived bearer token issued post-login |
```

## Synonyms

When the project uses multiple terms for the same concept, prefer one and list synonyms:

```markdown
| user | An authenticated principal. Synonyms: account, principal (avoid in spec text) | spec.md §3.1 |
```
